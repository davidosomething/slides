Divitis

Use semantic elements instead of DIVs: <header>, <footer>, <nav>, <h1>, <h2>, <blockquote>, <article>, <section>

Why?

It gives your page meaning, which some search engines use.

It makes sense to other developers.

It reduces page size (<div id="header"> is longer than <header>).

You don't need the <div> to style things, just target the thing inside the div (unless it's just text, then use a span or a paragraph because they make sense).

Writing great CSS

Rule #1

.indent { consistently: always !important; }
    #indent { consistently: always !important; }
    #indent { consistently: always !important; }
  #indent .consistently { }
   .indent.consistently { }
     .indent { consistently: always !important; }

Best practices

  • Use a normalizer -- see HTML5boilerplate's CSS
  • Use semantic class names: "error" instead of "red", "sidebar" instead of "rightcolumn"
  • Use the shortest selector (pagespeed)
  • Learn to use Firebug / Inspector / Developer Tools
