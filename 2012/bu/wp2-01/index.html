Review of WordPress Requirements

You need:

  1. A server to store all the WordPress files.
    The "A" for Apache in MAMP provides this.
  2. PHP on the server.
    That's the "P" in MAMP.
  3. A database for WordPress to store blog posts and settings.
    The 2nd "M" for MySQL in MAMP provides this.

Install WordPress (files)

To install WordPress, start with setting up the files and folders.

  1. Download a clean copy of WordPress.
  2. Unzip it if it is zipped.
  3. Rename the "wordpress" folder to whatever you want to call your site (e.g. "mysite").
    Only use lowercase letters
  4. Copy it from the Downloads folder (Student/Downloads) to:
    /Applications/MAMP/htdocs ("the htdocs folder")
  5. Start MAMP if it isn't running.
  6. Go to your your site on localhost, e.g.
    http://localhost:8888/mysite
  7. Follow the instructions, which ask you for a database.

Install WordPress (database)

To create a database:

  1. Open up the MAMP start page:
    http://localhost:8888/MAMP
  2. Go to phpMyAdmin
  3. Go to the privileges section.
  4. Add a new user.
  5. Enter in some username and password. (Write down somewhere, noting that these are for the database!)
  6. Use the dropdown to select "local" for the host.
  7. Check the box that says "create a database with the same name"
  8. Click "go" on the bottom of the page.

Install WordPress (admin)

Now that you have a database, you can continue with the WordPress install

  1. Click "Let's Go" in the WordPress Installer.
  2. Fill in the database info.
  3. Run the install.
  4. Enter in your WordPress info.
  5. Don't forget to change your username to something other than "admin"
  6. Take note of your WordPress username and password! It can be different from your database username and password.

Vocabulary review

  • <span class="awesome">I'm awesome!</span> — This is an HTML element. It has tags, content, and other hidden properties.
  • <span class="awesome">I'm awesome!</span> — This is an HTML start tag.
  • <span class="awesome">I'm awesome!</span> — This is the tag's name.
  • <span class="awesome">I'm awesome!</span> — This is an attribute.
  • <span class="awesome">I'm awesome!</span> — This is the attribute's value.
  • <span class="awesome">I'm awesome!</span> — This is the contents of the HTML element.
  • <span class="awesome">I'm awesome!</span> — This is the end tag for the span element.

Vocabulary review

  • nav a { text-decoration: none; color: blue; } — This is a CSS rule.
  • nav a { text-decoration: none; color: blue; } — This is the rule's selector.
  • nav a { text-decoration: none; color: blue; } — These are the style declarations for the rule.
  • nav a { text-decoration: none; color: blue; } — This is a style declaration's property.
  • nav a { text-decoration: none; color: blue; } — This is that property's value.

Vocabulary review

  • <?php the_title(); ?> — This is a PHP block.
  • <?php the_title(); ?> — This is a PHP start tag.
  • <?php the_title(); ?> — This is a function statement. It "calls" the function named "the_title".
  • <?php the_title(); ?> — This is a PHP end tag.

Child theming

What is it?

It's a theme based off of another one.

They use the same files, but you can override parent files by adding different versions in the child.

This lets you build off of an existing theme without modifying the existing files.

Why use it?

Already SEO optimized, has options panels, custom WP stuff.

If the parent updates its functions or CSS, you get all the updates.

Codex page

Where to start?

You can child theme off of ANY theme, even ones you made yourself.

Some nice, clean ones include:
